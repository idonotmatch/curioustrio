# New User Flow Mockups

These mockups now explore a much lighter first-run experience for Adlo.

The earlier version leaned too much toward:

- setup overview
- checklist management
- multiple tasks visible at once

The revised direction is intentionally simpler:

- one centered card at a time
- one question per step
- very short copy
- one clear next move

## Experience reference

The feel we are aiming for is closer to:

- Typeform
- a calm interview flow
- one meaningful choice at a time

Not:

- a setup dashboard
- a feature tour
- an admin checklist

## Shared principles

- keep the user focused on the current decision only
- do not show more than 2-3 choices at once
- make solo mode feel first-class
- keep optional steps truly optional
- let Summary carry a light nudge, not onboarding bulk

## Recommendation

The flow should behave like a short conversation:

1. how are you starting
2. what would help first
3. one optional follow-up
4. into the app

That is the right level of structure for a new user here.
